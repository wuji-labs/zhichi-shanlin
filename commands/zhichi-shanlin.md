# /zhichi-shanlin

手动触发 咫尺山林 skill。当一个空间「太平 / 太满 / 没有去处 / 用一次就不回来」时，输入 `/zhichi-shanlin` 进入「空间·境」审校模式。

`$ARGUMENTS` 是待审校的对象(一段界面/场景/路径描述，或一个文件/截图路径)。为空时，对当前上下文中正在做的空间任务施治。

## 触发后执行

1. **先看够**(身即山川 · 饱游饫看):先真正看清对象——它的近景/中景/远景各是什么、动线从哪进从哪出、焦点在哪、哪里是空的。不照模板套壳。
2. **诊三远**(高遠/深遠/平遠):问「纵深从哪来」。当前立了几种远?近—中—远是否各有交代?缺哪一种?
3. **判四可**(可行/可望/可游/可居):逐级体检——路通吗(可行)?有看头吗(可望)?逛得有奖赏吗(可游)?想久留想回来吗(可居)?指出止步在哪一级。
4. **查留白**(烟霞锁腰):空白是「还没填」还是「让山显高的那口气」?该藏的有没有藏一半?该空的有没有空出来?
5. **铺路**(步步移面面看):多屏/多步路径上,每一步看到的是否不同、是否有新发现?
6. **应时**(四时朝暮):loading/empty/active/night 等状态是否各有气象,还是一张静态贴图?

需要典源原文时读 [reference/linquan-gaozhi.md](../reference/linquan-gaozhi.md);引用须注「书·篇」,reference 之外不杜撰原句。

## 输出格式(固定状态行 + 三段)

先打一行状态:

```
[境 | 三远:立✓缺✗ | 止步级:可X | 主病:Y | 一招:Z]
```

然后三段:

### 一、看见了什么(现状的近—中—远 + 焦点 + 动线)
逐条列出当前空间的近景/中景/远景、焦点、进出动线、空白分布。只陈述,不评判。

### 二、诊断(对照四底层原则,标出止步在哪一级)
| 维度 | 现状 | 病灶 | 典源依据(书·篇) |
|------|------|------|----------------|
| 三远 | … | … | 林泉高致·山水训 |
| 四可 | 止于可X | … | 林泉高致·山水训 |
| 留白 | … | … | 林泉高致·山水训 |
| 游观/四时 | … | … | 林泉高致·山水训 |

### 三、重构处方(可落地的具体改法,按优先级)
1. **立远**:……
2. **求居**:……
3. **藏一半**:……
(每条给出「改前→改后」的具体差异,不要空泛口号。)
